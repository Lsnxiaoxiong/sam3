# SAM3 ONNX 跨图参考推理执行流程文档

## 1. 概述

### 1.1 功能描述
`infer_onnx_with_reference.py` 实现了基于 **RoI 特征传递** 的跨图参考推理机制。该脚本能够将第一张图像的分割标注特征提取并传递给后续图像，使后续图像的推理能够"参考"第一张图的视觉特征。

### 1.2 核心原理
```
┌─────────────────────────────────────────────────────────────────┐
│                    跨图参考推理流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  图像 1 → 编码 → 解码 → 检测 → 提取 RoI 特征 → 存入参考库       │
│                              ↓                                  │
│  图像 2 → 编码 → [融合参考特征] → 解码 → 检测                   │
│                              ↓                                  │
│  图像 3 → 编码 → [融合参考特征] → 解码 → 检测                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 关键优势
- **无需修改 ONNX 模型**：通过在语言特征层面进行特征融合
- **保持输入形状一致**：使用特征相加而非拼接，避免改变 token 数量
- **自动维度匹配**：自动检测并适配语言特征维度

## 4. 核心函数详解

### 4.1 `extract_roi_features()`

**作用**：从 backbone 特征图中提取感兴趣区域 (RoI) 的特征

**输入**：
| 参数 | 类型 | 形状 | 描述 |
|------|------|------|------|
| backbone_fpn | list[NDArray] | 3×[C, H, W] | FPN 特征图列表 |
| boxes_xywh | NDArray | [4] | 归一化框 [cx, cy, w, h] |
| image_size | tuple[int, int] | - | 原始图像尺寸 (width, height) |

**输出**：
| 返回 | 类型 | 形状 | 描述 |
|------|------|------|------|
| roi_features_pooled | NDArray | [1, C] | 全局平均池化后的 RoI 特征 |

**处理流程**：
```
┌─────────────────────────────────────────────────────────────────┐
│ 1. 坐标转换：cx,cy,w,h → xyxy                                   │
│    x1 = (cx - w/2) * image_width                                │
│    y1 = (cy - h/2) * image_height                               │
│    x2 = (cx + w/2) * image_width                                │
│    y2 = (cy + h/2) * image_height                               │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 2. 准备 RoI 框 (添加 batch 维度)                                    │
│    boxes_xyxy: [N, 5] = [[0, x1, y1, x2, y2]]                   │
│    (第一个元素是 batch index)                                    │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 3. RoI Align 池化                                                 │
│    - 使用 backbone_fpn[0] (最高分辨率特征图)                     │
│    - spatial_scale = 1008 / 72 = 14                             │
│    - output_size = (7, 7)                                       │
│    - sampling_ratio = 2                                         │
│    输出：[1, C, 7, 7]                                           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 4. 全局平均池化                                                  │
│    roi_features.mean(dim=(2, 3))                                │
│    输出：[1, C]                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 `augment_language_features()`

**作用**：将参考特征融合到语言特征中

**输入**：
| 参数 | 类型 | 形状 | 描述 |
|------|------|------|------|
| language_features | NDArray | [B, N, D] | 语言特征 (B=batch, N=tokens, D=dim) |
| reference_features | ReferenceFeature | - | 参考特征容器 |
| augment_type | str | - | 增强方式 ("add" | "replace_first") |

**输出**：
| 返回 | 类型 | 形状 | 描述 |
|------|------|------|------|
| augmented | NDArray | [B, N, D] | 增强后的语言特征 |

**处理流程（add 模式）**：
```
┌─────────────────────────────────────────────────────────────────┐
│ 输入：language_features [1, 32, 256]                            │
│       roi_features [1, C]                                       │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 1. 维度检查与匹配                                                │
│    if C != 256:                                                 │
│      - C < 256: zero-padding                                    │
│      - C > 256: truncate                                        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 2. 形状变换                                                      │
│    roi_feat: [1, C] → reshape → [1, 1, C]                      │
│    roi_feat_broadcast: tile → [1, 32, C]                       │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 3. 特征相加                                                      │
│    augmented = language_features + roi_feat_broadcast           │
│    输出：[1, 32, 256]                                           │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 `extract_reference_features()`

**作用**：从推理结果中提取最佳检测的特征作为参考

**处理流程**：
```
┌─────────────────────────────────────────────────────────────────┐
│ 1. 置信度过滤                                                    │
│    if max(scores) < threshold:                                  │
│      return None                                                │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 2. 选择最佳检测                                                  │
│    best_idx = argmax(scores)                                    │
│    best_box = boxes[best_idx]  # xyxy                           │
│    best_mask = masks[best_idx, 0]                               │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 3. 坐标转换                                                      │
│    xyxy → cx,cy,w,h                                             │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 4. 提取 RoI 特征并维度匹配                                         │
│    roi_features = extract_roi_features(...)                     │
│    if language_feature_dim is not None:                         │
│      padding/truncation                                         │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 5. 返回 ReferenceFeature                                        │
│    - roi_features: [1, D]                                       │
│    - boxes: [4]                                                 │
│    - masks: [H, W]                                              │
│    - scores: [1]                                                │
└─────────────────────────────────────────────────────────────────┘
```

## 5. 数据结构详解

### 5.1 `ReferenceFeature` Dataclass

```python
@dataclass
class ReferenceFeature:
    """存储从参考图像提取的特征"""
    roi_features: NDArray      # RoI 池化后的特征 [1, channels]
    boxes: NDArray             # 归一化框 [cx, cy, w, h]
    masks: NDArray             # 分割掩码 [1, H, W]
    scores: NDArray            # 置信度分数 [1]
```

### 5.2 特征维度对照表

| 特征名称 | 形状 | 来源 | 用途 |
|----------|------|------|------|
| backbone_fpn[i] | [C, H, W] | Image Encoder | 视觉特征提取 |
| vision_pos_enc[i] | [C, H, W] | Image Encoder | 位置编码 |
| language_features | [1, 32, 256] | Language Encoder | 文本语义表示 |
| language_mask | [1, 32] | Language Encoder | 文本注意力掩码 |
| roi_features | [1, C] | RoI Align | 参考区域特征 |
| augmented | [1, 32, 256] | augment_language_features | 融合后特征 |

### 5.3 ONNX 模型 I/O 规格

#### Image Encoder
| 输入/输出 | 名称 | 形状 | 类型 |
|-----------|------|------|------|
| 输入 | image | [1, 3, 1008, 1008] | float32 |
| 输出 | vision_pos_enc | 3×[1, 256, 72, 72] | float32 |
| 输出 | backbone_fpn | 3×[1, 256, H, W] | float32 |

#### Language Encoder
| 输入/输出 | 名称 | 形状 | 类型 |
|-----------|------|------|------|
| 输入 | tokens | [1, 32] | int64 |
| 输出 | language_mask | [1, 32] | float32 |
| 输出 | language_features | [1, 32, 256] | float32 |

#### Decoder
| 输入/输出 | 名称 | 形状 | 类型 |
|-----------|------|------|------|
| 输入 | original_height | [1] | int64 |
| 输入 | original_width | [1] | int64 |
| 输入 | backbone_fpn_0/1/2 | [1, 256, H, W] | float32 |
| 输入 | vision_pos_enc_2 | [1, 256, 72, 72] | float32 |
| 输入 | language_mask | [1, 32] | float32 |
| 输入 | language_features | [1, 32, 256] | float32 |
| 输入 | box_coords | [1, 1, 4] | float32 |
| 输入 | box_labels | [1, 1] | int64 |
| 输入 | box_masks | [1, 1] | bool |
| 输出 | boxes | [N, 4] | float32 |
| 输出 | scores | [N] | float32 |
| 输出 | masks | [N, 1, H, W] | float32 |

## 6. 使用指南

### 6.1 基本用法

```bash
# 单图推理
python infer_onnx_with_reference.py --image input.jpg --text-prompt "dog"

# 多图推理（不使用参考）
python infer_onnx_with_reference.py --images img1.jpg img2.jpg --text-prompt "keyboard"

# 多图推理（使用第一张图作为参考）
python infer_onnx_with_reference.py --images img1.jpg img2.jpg img3.jpg --text-prompt "keyboard" --use-reference
```

### 6.2 交互式模式

```bash
# 手动选择参考框
python infer_onnx_with_reference.py --images img1.jpg img2.jpg --text-prompt "target" --use-reference --interactive
```

### 6.3 参数调优

```bash
# 调整参考特征提取的置信度阈值
python infer_onnx_with_reference.py --images img1.jpg img2.jpg --text-prompt "dog" --use-reference --refine-threshold 0.7
```

### 6.4 框提示格式

```bash
# 使用归一化的 cx, cy, w, h 格式
python infer_onnx_with_reference.py --image input.jpg --box-prompt "0.5,0.5,0.2,0.2"
```

## 7. 性能特征

### 7.1 内存使用

| 阶段 | 峰值内存 | 说明 |
|------|----------|------|
| 模型加载 | ~2GB | 三个 ONNX 模型 |
| 图像编码 | ~500MB | backbone_fpn 缓存 |
| 参考特征提取 | ~100MB | RoI 特征池化 |
| 特征增强 | ~50MB | 语言特征处理 |

### 7.2 时间分析

| 操作 | 耗时 (典型值) |
|------|---------------|
| 图像编码 | ~500ms |
| 语言编码 | ~10ms |
| 解码 | ~200ms |
| RoI 特征提取 | ~50ms |
| 特征增强 | ~5ms |

### 7.3 优化建议

1. **批量处理**：多张图像共享一次语言编码
2. **特征缓存**：参考特征可复用于多次推理
3. **GPU 加速**：使用 CUDAExecutionProvider

## 8. 故障排除

### 8.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 维度不匹配错误 | RoI 特征维度与语言特征不匹配 | 检查 language_feature_dim 自动检测 |
| 参考特征提取失败 | 检测分数低于阈值 | 降低 --refine-threshold |
| 第二张图 OOM | GPU 内存不足 | 减小 batch 或使用 CPUExecutionProvider |
| 参考效果差 | 第一张图检测质量低 | 使用 --interactive 手动选择 |

### 8.2 调试模式

```bash

export LOGURU_LEVEL=DEBUG

```

## 9. 扩展方向

### 9.1 可能的改进

1. **多参考特征**：支持从多张图提取参考特征
2. **注意力融合**：使用注意力机制替代简单相加
3. **在线更新**：动态更新参考特征库
4. **类别感知**：针对不同类别使用不同参考特征

### 9.2 应用场景

- **视频目标跟踪**：将第一帧的标注传递给后续帧
- **批量图像处理**：使用样张指导批量分割
- **少样本学习**：单样本指导新图像分割
